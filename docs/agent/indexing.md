# 에이전트 인덱싱 가이드

## 목적
- 에이전트 컨텍스트를 작게 유지하고, 작업 범위에 맞는 문서만 로드한다.

## 적용 조건 (When to load)
- 항상 먼저 로드한다.
- 어떤 모듈 문서를 열어야 할지 결정할 때 로드한다.

## 기본 절차
1. 루트 `AGENTS.md`를 읽는다.
2. 변경 대상 Gradle 모듈을 식별한다.
3. 대상 모듈의 `AGENTS.md`만 추가 로드한다.
4. 다중 모듈 변경이면 변경되는 모듈 문서만 추가한다.
5. 구현/테스트/릴리즈 관련 세부 규칙이 필요할 때만 정책/품질/플레이북 문서를 추가 로드한다.

## 로드 최소화 규칙
- 계획 단계에서는 루트 + 대상 모듈 문서만 유지한다.
- 코딩 시작 시 필요한 섹션만 선택 로드한다.
- 작업과 무관한 모듈 `AGENTS.md`는 열지 않는다.
- 동일 규칙 재확인이 필요할 때는 원문 재로드 대신 작업 노트에 요약을 재사용한다.
