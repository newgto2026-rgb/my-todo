# PRD v0.2 - 카테고리 제거 + 중요도(낮음/중간/높음) 도입

- 작성일: 2026-04-15
- 버전: v0.2 (User-Centered)
- 상태: Draft

## 1) 문제 정의
현재 앱은 카테고리 입력/관리 비용 대비 사용자의 즉시 행동(무엇을 먼저 할지 결정) 지원이 약하다. 사용자는 분류보다 우선순위 판단을 더 빠르게 원한다.

## 2) 목표
- 할 일 입력 시간을 단축한다.
- 사용자에게 오늘 처리할 핵심 항목을 즉시 보이게 한다.
- 완료/미완료/중요도 상태를 일관된 톤으로 보여준다.

## 3) 성공 지표
- 할 일 생성 완료 시간 20% 감소
- 높은 중요도 항목 당일 완료율 상승
- 중요도 필터 사용률 상승
- 중요도 변경 후 Undo 사용 흐름 정상(오동작/오입력 감소)

## 4) 핵심 사용자 시나리오
1. 빠른 추가 사용자: 기본값(중간)으로 빠르게 저장
2. 계획형 사용자: 생성/수정 시 중요도 지정 후 높은 중요도 우선 처리
3. 당일 실행 사용자: Today/Calendar에서 높은 중요도 위주로 확인

## 5) 기능 요구사항
### 5.1 카테고리 기능 제거
- 카테고리 필터/관리/색상/아이콘/편집 UI 제거
- 카테고리 관련 문구 및 빈 상태 문구 정리

### 5.2 중요도 도입
- 값: `LOW`, `MEDIUM`, `HIGH`
- 표시명: `낮음`, `중간`, `높음`
- 기본값: `중간`
- 생성/수정 화면에서 단일 선택

### 5.3 목록/정렬/필터
- 기본 정렬: 미완료 우선 -> 중요도(HIGH > MEDIUM > LOW) -> 마감일/시간
- 필터: 전체/높음/중간/낮음
- 필터 상태 재진입 시 유지

### 5.4 캘린더 반영
- 날짜 상세 리스트에서 중요도 표시
- 같은 날짜 내 정렬은 중요도 우선

### 5.5 Undo
- 중요도 변경/완료 토글 후 스낵바 Undo 제공

## 6) UX 가이드
- 릴리스 후 1회 안내: 카테고리가 중요도로 단순화되었고 기존 항목은 중간으로 이관됨
- 중요도 설명 보조 문구
  - 높음: 오늘 놓치면 안 되는 일
  - 중간: 이번 주 내 처리
  - 낮음: 여유 있을 때 처리
- 색상만으로 구분하지 않고 텍스트 배지/아이콘 병행

## 7) 데이터/도메인 요구사항
- Todo 엔티티에 `priority` non-null 컬럼 추가, default `MEDIUM`
- 마이그레이션 시 기존 데이터는 `MEDIUM`으로 이관
- enum 저장은 문자열 기반
- 카테고리 의존 로직/조회 제거

## 8) 비기능 요구사항
- 데이터 손실 없는 마이그레이션
- 리스트/캘린더 성능 유지
- 접근성 대비 준수

## 9) 수용 기준(AC)
- 카테고리 UI가 앱 전역에서 제거됨
- 새 항목 기본 중요도 `중간`
- 중요도 변경 즉시 반영(목록/Today/Calendar)
- 기존 데이터 이관 후 모두 `중간`으로 표시
- 필터 상태 유지
- Undo 정상 동작

## 10) 테스트 범위
- 단위: 정렬/필터/매핑/Undo
- 마이그레이션: old -> new 스키마 데이터 보존 검증
- UI: 생성/수정/필터/Today/Calendar 주요 플로우

## 11) 리스크
- 카테고리 제거에 대한 일부 사용자 반발 가능
- 대응: 변경 안내 + 향후 태그/프로젝트 확장 가능성 백로그 관리

## 12) 릴리스 노트
카테고리 대신 중요도(낮음/중간/높음)로 단순화하여, 오늘 해야 할 일을 더 빠르게 우선순위로 확인할 수 있다.
