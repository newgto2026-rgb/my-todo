# :core:ui 모듈 가이드

## 역할
- 여러 기능에서 재사용되는 UI 컴포넌트.

## 규칙
- 가능한 한 상태 없는(stateless) 컴포넌트로 유지한다.
- 기능 전용 비즈니스 로직을 넣지 않는다.
- 디자인 시스템 토큰/스타일을 따른다.
- 공용 UI 컴포넌트는 전달받은 상태를 표현한다. domain model을 직접 해석해 서비스 정책을 판단하지 않는다.
- View/Composable에는 프레젠테이션 로직, 비즈니스 로직, 애플리케이션 로직을 추가하지 않는다.
- 화면 문구, 접근성 라벨, 버튼 활성화, 상태별 노출 여부는 호출 측의 `UiState`/파라미터로 전달받는다.

## 변경 체크리스트
- 널리 쓰이는 컴포저블의 바이너리 호환성을 유지한다.
- 해당 시 접근성 라벨과 테스트 태그를 점검한다.

## 검증
- `./gradlew :core:ui:lintDebug`
